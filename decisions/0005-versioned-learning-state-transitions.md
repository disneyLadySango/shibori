# ADR-0005: バージョン付き学習状態と明示的な遷移

- Status: accepted
- Date: 2026-07-20

## Context

探索、現在地、集中先、理解確認、抜け、補強、再開は同じ学習の一周である。画面ごとに別々の事実を持つと、教材へ触れただけで理解済みになる、補強後に戻れない、再開時に状態を推測するといった原点との不一致が起きる。

## Decision

- 学習状態をバージョン付きの一つの構造として保持する。
- 理解状態を変える操作を純粋な状態遷移として実装する。
- 教材への接触は状態遷移に含めず、理解確認の結果だけが理解状態を変える。
- 永続化した状態は復元時にも同じスキーマで検証する。

## Consequences

- 17の承認済みテスト仕様を、UIや外部モデルから独立して検証できる。
- 将来スキーマを変える場合はバージョン移行が必要になる。
- 生成モデルは提案と判定を担うが、保存状態の整合性はアプリケーションが担う。
