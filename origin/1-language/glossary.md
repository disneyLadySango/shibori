# ユビキタス言語

- state: `proposed`
- revision: `5`
- updated: `2026-07-20`
- approved_by: 未設定

下位層は、重要概念についてこの文書の語彙だけを使う。

| 用語 | 定義 | 含めないもの |
| --- | --- | --- |
| 学習者 | 何かを理解または実行できる状態へ変わろうとして学ぶ人 | 自律性、年齢、所属による限定 |
| 学習目的 | 学習者が、なぜ学びたいのかという理由や欲求。知りたい、面白いという欲求だけでも成立する | 実用性、期限、優先度、何をできるようになるかという到達状態 |
| 到達状態 | 学習によって何を理解・説明・実行できるかを「〜できる」というCanの形で表した状態 | 「〜したい」というWill、学ぶ理由、教材の修了、学習量 |
| 学習領域 | 一つの到達状態に向けて知識と実践を積み上げるまとまり | 単なるコンテンツ分類 |
| 学習経路 | 現在の理解状態から到達状態までに必要な理解と実践の依存関係、および現在地 | 固定された教科書の目次 |
| 集中資源 | 学習者が一定期間に使える、注意の深さと時間の組み合わせ | 総可処分時間だけ |
| 深い集中 | 計算、記述、判断、想起など、能動的に手を動かす注意状態 | 長時間であること自体 |
| 軽い集中 | 移動や家事と並行しながら概念や関係をつかめる注意状態 | 理解を確認せず聞き流すこと |
| 集中先 | 現在の集中資源を優先的に投じる、ただ一つの理解または実践 | 興味のある候補一覧 |
| 推奨 | 学習目的、到達状態、理解状態、集中資源から、次の集中先として最も適すると判断した一つを理由とともに示すこと | 命令、学習者に代わる最終決定、候補の羅列 |
| 配分 | 学習目的、到達状態、理解状態、集中資源に基づき、何へどの深さの注意を使うか決めること | 教材を均等に消化すること |
| 理解状態 | 学習者が今、何を理解・説明・実行でき、何がまだ難しいか | 閲覧時間、再生完了、教材の進捗だけから決めた状態 |
| 理解確認 | 説明、計算、判断、想起などを通じて、現在の理解状態を確かめること | 教材を読んだ、聞いたという行為だけ |
| 確認結果 | 理解確認によって分かった、できたこととまだできないこと | 学習者の能力を固定的に評価する成績 |
| 抜け | 到達状態に必要な前提のうち、確認結果から、まだ理解・説明・実行できないと分かったこと | 未確認の前提、単なる苦手意識 |
| 補強 | 抜けを解消するため、一時的に学習経路の優先順位を変えること。抜けが見つかった場合は推奨するが、取り組む時期は学習者が決める | 失敗への罰、強制、全体のやり直し、今取り組まないことを理由に他の学びを妨げること |
| 再配分 | 新しい確認結果または抜けに応じ、集中先と学習経路を更新すること | 毎回すべてを作り直すこと |
| 学習ポートフォリオ | 学習者が並行して持つ学習領域と、それらへの集中資源の配分全体 | 成績や資格の一覧 |
| 今日 | 学習者が次に利用できる一回の軽い集中または深い集中の機会 | 暦日の0時から24時に限定しない |

## 追加提案が必要な語

- 「習得」は理解状態と到達状態のどちらとも境界が曖昧なため、判定条件が定義されるまで下位層で使わない。
- 「撤退」を、学習領域の停止とプロダクト仮説の停止のどちらにも使うと曖昧になるため、承認時に分離を検討する。

## Revision history

- revision 1, 2026-07-20: 既存射影から逆順蒸留。学習者の定義から「自律」と「複数の責務」を除外。
- revision 2, 2026-07-20: 学ぶ理由である「学習目的」と、できるようになりたい「到達状態」を分離。「理解証拠」を廃止し、理解状態、理解確認、確認結果へ分解。
- revision 3, 2026-07-20: 学習目的は知りたいという欲求だけでも成立し、実用性、期限、優先度を必須にしないと確定。到達状態をWillではなくCanで表すと確定。
- revision 4, 2026-07-20: 推奨を理由つきの最有力案かつ非強制と定義。抜けの補強は推奨するが、取り組む時期は学習者が決めると確定。
- revision 5, 2026-07-20: 補強へ今取り組まない場合も他の学びを妨げず、教材へ触れたことだけでは理解状態や抜けを変更しない制約をQ-004から導出。
