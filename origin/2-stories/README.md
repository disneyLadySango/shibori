# Stories index

第2層は課題の正本であり、具体策を置かない。異なる性質の記録を一つの文書へ混ぜない。

## 構成

- `story-map.md`: すべてのユーザーストーリーを統括する唯一のマップ。活動順、優先順位、リリーススライスを持つ
- `backlog/US-xxx.md`: 発見済みだが、まだリリースへ選択していないユーザーストーリー
- `active/US-xxx.md`: 現在のリリーススライスで検証するユーザーストーリー
- `archive/YYYY/US-xxx.md`: 完了、撤退、または課題が偽と判明したユーザーストーリー
- `evidence/`: 課題を支持または反証する観測事実
- `history/`: 過去に試した具体策の結果と学び
- `open-questions.md`: 人間が所有する未決定の意図

## 更新規則

1. 観測が得られたら、まず`evidence/`へ追加する。
2. 観測が既存ストーリーを支持・反証する場合、ストーリー側へevidence IDを追加する。
3. 新しいストーリーは既存語彙で表現できる場合だけ`backlog/`へ追加し、`story-map.md`へ配置する。
4. 具体策が失敗してもストーリーは削除せず、`history/`へ結果を残す。
5. 課題自体が偽と確認された場合だけ、ストーリーを`retired`にする。
6. 各ユーザーストーリーは第3層に一つ以上のACを持つ。所有関係はUS-IDで表す。
7. リリーススライスへ選択したストーリーだけを`active/`へ移す。ストーリーマップ上の配置だけでactive扱いしない。
