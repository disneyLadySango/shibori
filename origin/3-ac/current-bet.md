# 現在の賭けと成果基準 — 初期ドラフト

状態: **人間承認待ち**

## 賭け

複数の学習目的を持つ学習者に対し、学習経路上の現在地、次の集中先、その理由を一つに絞って提示し、理解証拠と抜けによって再配分すれば、教材選択と計画修正に使う認知負荷を減らし、希少な深い集中を目的に直結する実践へ使える。

## 今回のHowに紐づくAC

### AC-01 学習ポートフォリオの分離

- 学習者は二つ以上の学習領域を、それぞれ異なる学習目的とともに保持できる。
- ある領域の現在地、理解証拠、抜けが、別領域へ混入しない。
- どの領域へ集中資源を配分しているか常に一意である。

### AC-02 全体と現在地の理解

- 学習者は、選択した領域について学習目的までの学習経路を順序付きで把握できる。
- 現在地は一つだけであり、前後の依存関係と区別できる。
- 「なぜ今ここか」が学習目的と最近の理解証拠または抜けに結びついている。

### AC-03 今日の選択と集中

- 次の軽い集中と深い集中について、それぞれ行うことが最大一つに絞られる。
- 学習者は候補比較をせず、利用可能な注意状態に合う行動を開始できる。
- 深い集中先は、閲覧ではなく理解証拠を生む行為である。

### AC-04 理解による再配分

- 学習者が理解証拠を示すか抜けを報告すると、同じ学習領域の現在地または次の集中先が再評価される。
- 変更前後の差と変更理由を学習者が確認できる。
- 抜けを補強しても、元の学習目的と復帰先を見失わない。

### AC-05 継続性

- 中断後、学習者は以前の判断を再現する作業なしに、選択中の領域、現在地、次の集中先へ戻れる。
- 未完了の理解証拠と、確認済みの理解証拠を区別できる。

### AC-06 判断の誠実性

- 理解証拠がない内容を、消費完了だけで習得済みにしない。
- 推定、デモ、実際の理解証拠を学習者が区別できる。
- 学習経路または集中先を変更できない場合、古い判断を新しい判断として見せない。

### AC-07 Build Week制約

- すべての言語モデル判断はGPT-5.6を利用する。
- 審査者が、GPT-5.6が配分、学習経路、理解評価へ与える役割を公開成果物から追跡できる。
- AI生成音声は人間の音声と誤認されない。

## ガードレール

以下は価値仮説の成功に関係なく破ってはならない。

1. 学習領域間で目的、理解証拠、抜けを混同しない。
2. 一つの注意状態に対し、同格の「今やること」を複数提示しない。
3. 学習者の入力や理解状態を、本人に知らせず公開または第三者共有しない。
4. 自動判断の根拠を、学習者の目的または観測された理解状態へ結びつける。
5. 外部サービスが利用できない場合、固定結果を任意入力への実判断として見せない。

## 成功条件の提案値 — 人間承認が必要

以下の数値はエージェントによる検証可能性の提案であり、未承認のため確定条件ではない。

1. 対象学習者5人中4人以上が、初見で30秒以内に「今集中する一つ」と「なぜそれか」を説明できる。
2. 5人中4人以上が、二つの学習領域を切り替えた後も、それぞれの現在地を正しく説明できる。
3. 抜けが一件判明したシナリオで、5人中4人以上が変更された集中先と元の目的への復帰関係を説明できる。
4. 再訪シナリオで、5人中4人以上が15秒以内に次の行動を開始できる。

## 撤退条件の提案値 — 人間承認が必要

1. 二回の改善サイクル後も、対象学習者の過半数が「今やる一つ」を自力で特定できない場合、現在の配分表現を畳む。
2. 対象学習者の過半数が、自分で教材や順番を選ぶ方が認知負荷が低いと一貫して評価した場合、自動配分の賭けを畳む。
3. 抜けによる再配分が、過半数の学習者に「遠回りさせられた」と受け取られ、理由提示を変えても改善しない場合、補強を現在地へ強制する賭けを畳む。
4. 複数領域の同時保持が、単一領域利用より集中の分散を増やす場合、領域間配分の賭けを畳み、単一領域へ範囲を戻す。

## スコープ

### IN候補

- Slice A「次の一回に迷わない」全体。
- Slice Bのうち、複数領域の分離と一つの選択中領域。
- 理解証拠または抜けによる現在地の変更と理由説明。
- 中断後の再開。

### OUT候補

- 複数人の共同学習、教師向け管理。
- 課金、組織認証、公開プロフィール。
- 教材市場、教材推薦の網羅性。
- 学習時間の最大化や連続日数のゲーミフィケーション。
- 長期的効果が未検証のまま行う完全自動の領域間優先順位決定。
